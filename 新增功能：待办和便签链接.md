
## PRD：待办事项 ↔ 便签关联（Task–Note Linking）

### 1. 背景与目标
你现在的使用场景是：  
- 做任务时临时需要记录过程/灵感/材料  
- 希望**在任务列表里就能快速新建一条“与该任务关联”的便签**  
- 或者先写便签，之后再把它**关联到某个任务**  
目标是让“任务”成为入口，“便签”承载内容，同时二者可互相跳转。

### 2. 术语与范围
- **Task**：待办事项（现有）
- **Note**：便签（现有）
- **Link/关联**：Task 与 Note 之间的一条关系（新增）
- **关联类型**：默认 1 个 Task 可关联多条 Note；1 条 Note 也可关联多个 Task（可选，见 6.1）

本 PRD 默认采用：**多对多（Task ↔ Note）**，因为最灵活、后期不会卡死需求。

### 3. 用户故事（User Stories）
- **US-1（任务侧快速记录）**  
  作为用户，我在任务列表中对某个任务右键/按钮点击“关联便签”，可立即新建便签并自动绑定到该任务。
- **US-2（任务侧查看记录）**  
  作为用户，我希望在任务卡片上看到“已关联便签数量/图标”，点击后能看到关联便签列表并跳转。
- **US-3（便签侧绑定任务）**  
  作为用户，我写好一条便签后，能在便签界面把它关联到某个任务（搜索任务/最近任务）。
- **US-4（解除关联）**  
  作为用户，我能解除某条便签与某个任务的关联，不影响便签本体内容。
- **US-5（删除与回收站规则清晰）**  
  作为用户，我删除任务/删除便签时，关联关系自动清理，但不会误删对方数据。

### 4. 核心功能需求（Functional Requirements）

#### 4.1 任务列表入口（最重要）
- **FR-1**：任务项右键菜单新增：
  - `📝 新建关联便签`
  - `🔗 关联已有便签…`
  - `📎 查看关联便签 (N)`
- **FR-2**：`新建关联便签`
  - 立即创建 Note（标题可自动填充为任务标题前缀，例如：`[任务] xxx`，可配置）
  - 自动建立关联
  - UI 行为（可选其一，建议 A）  
    - A：直接打开便签页并加载该新便签（用户开始写）  
    - B：弹出小浮层/浮动便签窗口（更像“随手记”）
- **FR-3**：`查看关联便签 (N)`
  - 弹出一个小面板（类似回收站那种 EditorDialog 或轻量 popup）
  - 展示关联便签列表（标题 + 更新时间 + 跳转按钮 + 解除关联按钮）
- **FR-4**：任务卡片右侧可选新增一个小图标（📎/📝）与数量：`2`
  - 点击同 `查看关联便签`

#### 4.2 便签侧入口
- **FR-5**：便签编辑区域新增“关联任务”入口（例如标题栏右侧一个 `🔗` 按钮）
- **FR-6**：点击后弹窗：
  - 搜索任务（支持关键字）
  - 最近编辑/最近完成/今日任务快捷区
  - 已关联任务列表（可一键解除）
- **FR-7**：支持从便签跳转到任务：
  - 点击关联任务条目后，主窗口切换到任务页并定位/高亮该任务（如果当前已有定位能力；没有就先实现“切换到包含该任务的筛选视图 + 滚动到可见”）

#### 4.3 关联规则与一致性
- **FR-8**：关联关系是“轻量引用”，不复制内容。
- **FR-9**：删除行为
  - 删除任务：只删除 Task，**自动删除所有关联关系**；Note 不受影响
  - 删除便签（软删除进回收站）：关联关系保留还是隐藏？建议：
    - **建议方案**：便签软删除后在“关联便签列表”里不显示（默认过滤 deleted=True），但关系可保留；便签恢复后仍关联
  - 便签永久删除：自动清理关联关系
- **FR-10**：空便签规则延续：与任务关联的新建便签如果没内容，仍不应落库/显示（你之前的规则继续生效）

### 5. 非功能需求（NFR）
- **NFR-1**：操作路径不超过 2 步（任务→新建关联便签：1 步）
- **NFR-2**：不引入明显卡顿：关联查询需有索引/高效查询
- **NFR-3**：兼容旧数据：老任务/老便签无关联时不影响任何功能

## 需要你确认的 2 个关键产品选择（决定实现复杂度）
- **[选择 1]** 关联关系是否允许 **Note ↔ 多 Task**？  
  - **建议：允许多对多**（未来更灵活）

## 10. 验收标准（Acceptance Criteria）

### 10.1 任务侧功能验收
- **AC-1**：任务右键菜单显示"📝 新建关联便签"、"🔗 关联已有便签…"、"📎 查看关联便签 (N)"（N>0时显示数量）
- **AC-2**：点击"📝 新建关联便签"后：
  - 立即弹出浮动便签窗口
  - 标题自动填充为 `[任务标题] `（前缀+空格）
  - 便签与任务自动建立关联
  - 浮动窗口关闭时，任务卡片右侧显示关联数量+1
- **AC-3**：点击"🔗 关联已有便签…"后：
  - 弹出便签搜索/选择弹窗
  - 支持关键字搜索便签
  - 支持多选（Ctrl+点击/Shift+点击）
  - 确认后建立关联，关闭弹窗
- **AC-4**：点击"📎 查看关联便签"后：
  - 弹出关联便签列表
  - 显示便签标题、更新时间、操作按钮
  - 支持搜索、打开、解除关联
  - 支持新建关联便签

### 10.2 便签侧功能验收
- **AC-5**：便签编辑页标题栏显示"🔗 关联"按钮
- **AC-6**：点击"🔗 关联"后：
  - 弹出关联管理弹窗
  - 显示已关联任务列表（任务标题、状态、操作按钮）
  - 支持搜索任务
  - 支持添加新关联
  - 支持解除关联
- **AC-7**：点击关联任务条目后：
  - 主窗口切换到任务页
  - 自动定位并高亮该任务（如果支持定位）
  - 否则切换到包含该任务的筛选视图并滚动到可见

### 10.3 数据一致性验收
- **AC-8**：多对多关系正确建立和查询
- **AC-9**：删除任务后，所有相关关联关系自动清理，便签不受影响
- **AC-10**：删除便签（软删除）后，关联关系保留但不在列表显示；恢复便签后关联关系恢复显示
- **AC-11**：永久删除便签后，关联关系自动清理

### 10.4 性能与兼容性验收
- **AC-12**：关联查询响应时间 < 200ms（1000条便签+1000条任务）
- **AC-13**：旧数据（无关联的任务/便签）正常显示，不影响现有功能
- **AC-14**：空便签规则继续生效：关联的新建便签无内容时不落库/不显示

## 11. 边界用例（Edge Cases）

### 11.1 数据边界
- **EC-1**：任务关联大量便签（>100条）时的分页或虚拟滚动
- **EC-2**：便签关联大量任务（>100条）时的分页或虚拟滚动
- **EC-3**：重复关联同一任务/便签的处理（应阻止或提示已存在）
- **EC-4**：不适用说明：Task↔Note 为异构关系，不存在“循环依赖”概念；只需防重复关联

### 11.2 用户操作边界
- **EC-5**：快速连续点击"新建关联便签"的处理（防抖或阻止重复创建）
- **EC-6**：浮动便签窗口未保存就关闭的处理（询问保存或直接丢弃）
- **EC-7**：搜索结果为空时的友好提示
- **EC-8**：本地数据库异常时的错误处理（本应用离线，无网络依赖）

### 11.3 界面边界
- **EC-9**：任务标题过长时的截断显示
- **EC-10**：便签标题过长时的截断显示
- **EC-11**：关联数量过多时的显示限制（如显示"99+"）
- **EC-12**：小屏幕分辨率下的弹窗适配

## 12. 埋点与配置项（Analytics & Configuration）

> 备注：当前项目是本地离线桌面应用，埋点应作为可选项（默认关闭），避免增加首版复杂度与维护负担。

### 12.1 用户行为埋点
- **Event-1**：`task_note_link_create` - 创建关联关系
  - 属性：source（task_side/note_side）、task_id、note_id
- **Event-2**：`task_note_link_delete` - 删除关联关系
  - 属性：source（task_side/note_side）、task_id、note_id
- **Event-3**：`task_note_link_view` - 查看关联列表
- **Event-4**：`task_note_link_navigate` - 从便签跳转到任务
  - 属性：note_id、task_id
- **Event-5**：`task_note_link_navigate_reverse` - 从任务跳转到便签
  - 属性：task_id、note_id

### 12.2 配置项
- **Config-1**：`note_link_title_prefix` - 新建关联便签时的标题前缀
  - 默认值：`"[任务标题] "`
  - 可配置：支持自定义模板，如`"{task_title} - "`
- **Config-2**：`note_link_auto_save` - 浮动便签窗口关闭时自动保存
  - 默认值：`true`
  - 可配置：`false`（关闭时询问）
- **Config-3**：`note_link_max_display_count` - 关联数量显示上限
  - 默认值：`99`
  - 可配置：任意正整数
- **Config-4**：`note_link_search_debounce_ms` - 搜索防抖时间
  - 默认值：`300`
  - 可配置：0-2000ms

## 15. 评审补充：未覆盖点 & 复杂度降级建议（已整理）

### 15.1 文档里尚未明确、但实现时容易踩坑的点
- **GAP-1：数据迁移策略**
  - 需要明确 `task_note_links` 的建表迁移时机、旧版本兼容、失败回滚策略。
- **GAP-2：并发与一致性**
  - 同时在任务侧和便签侧操作关联时，是否需要乐观刷新/冲突处理（例如一边解除、一边打开）。
- **GAP-3：跨窗口同步**
  - 已存在主窗口 + 浮动便签窗口；关联变更后，任务卡片计数与便签关联列表需即时刷新。
- **GAP-4：软删除与过滤规则统一**
  - “关联列表不显示软删除便签”应在服务层统一过滤，避免 UI 各处重复写条件。
- **GAP-5：回收站恢复后的一致性**
  - 便签恢复后，关联数量与列表是否自动回补、何时刷新，需要明确触发点。
- **GAP-6：定位失败兜底文案**
  - 便签跳任务时若任务已删除/不可见，需统一 toast 文案与回退行为。
- **GAP-7：权限/单实例交互影响**
  - 关联功能会打开/切换窗口，需与现有单实例唤醒逻辑联测，避免“跳转触发新实例”的回归。

### 15.2 目前偏复杂、建议降级到后续版本的项
- **COMPLEX-1：首版即支持多选关联已有便签（Ctrl/Shift）**
  - 建议首版先单选 + 连续操作，减少 UI 复杂度。
- **COMPLEX-2：首版即做全量埋点 + 性能指标面板**
  - 建议推迟到 P3，先保留日志级别调试。
- **COMPLEX-3：任务/便签两侧都做完整搜索 + 最近区 + 分页**
  - 建议首版先做“关键词包含匹配 + 前 50 条”，分页后置。
- **COMPLEX-4：首版即实现精准定位 + 高亮 + 自动筛选切换**
  - 建议首版先“切换到任务页并尽量滚动可见”，高亮动画后置。

### 15.3 建议的“更稳 MVP”范围（两周内更容易落地）
- **MVP-S1（必须）**：
  - 任务右键 `📝 新建关联便签`
  - 自动创建关联
  - 任务卡片显示 `📎N`
  - 任务侧“查看关联便签”列表（仅打开/解除）
- **MVP-S2（建议）**：
  - 便签侧 `🔗 关联任务`（仅单选关联 + 解除）
- **MVP-S3（延期）**：
  - 多选关联、分页、埋点、复杂跳转高亮

### 15.4 建议新增验收条目（补足回归风险）
- **AC-15（迁移）**：升级后老数据可正常启动，且不影响原有任务/便签显示。
- **AC-16（同步）**：在浮动便签窗口解除/新增关联后，任务列表计数在 1 秒内更新。
- **AC-17（容错）**：目标任务或便签被删除时，打开关联项给出提示且不崩溃。
- **AC-18（去重）**：重复关联请求不会产生重复行，UI 给出“已关联”反馈。
- **AC-19（回收站）**：软删除便签后任务侧列表隐藏；恢复后可见且计数恢复。
