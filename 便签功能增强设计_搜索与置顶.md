# 便签功能增强设计：搜索与独立窗口置顶

## 一、功能概述

本次增强包含两个独立功能点：
1. **便签列表搜索功能**：在主窗口便签列表中快速查找便签
2. **独立便签窗口置顶按钮**：每个拖出的便签窗口可独立控制置顶状态

---

## 二、功能1：便签列表搜索

### 2.1 需求分析

**核心需求**：
- 用户可以在便签列表中快速搜索便签（按标题或正文内容）
- 搜索结果实时过滤显示
- 搜索框应易于访问且不占用过多空间

**使用场景**：
- 便签数量较多时（10+），需要快速定位特定便签
- 记不清便签标题，但记得部分关键词
- 想查看包含某个主题的所有便签

### 2.2 设计方案

#### 2.2.1 UI布局

**位置选择**：在便签列表面板的顶部（拖动手柄和"便签列表"标题之间）

**布局结构**：
```
┌─────────────────────────────────────┐
│ ≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡ │ ← 拖动手柄（8px）
├─────────────────────────────────────┤
│ 🔍 [搜索便签...]            [✕]     │ ← 搜索框（28px高）
├─────────────────────────────────────┤
│ 便签1                    03-27  🗑  │ ← 直接显示便签列表
│ 便签2                    03-26  🗑  │
│ ...                                 │
└─────────────────────────────────────┘
```

**设计说明**：
- 去除"▼ 便签列表"标题行，界面更简洁
- 搜索框下方直接显示便签列表
- 拖动手柄作为视觉分隔，无需额外标题

**搜索框设计**：
- 左侧：🔍 图标（固定，不可点击，仅作视觉提示）
- 中间：输入框，placeholder="搜索便签..."
- 右侧：清空按钮 ✕（仅在有输入时显示）
- 高度：28px
- 内边距：左右12px，上下4px
- 背景：半透明，与主题融合

#### 2.2.2 交互逻辑

**搜索触发**：
- 实时搜索：每次输入变化后延迟300ms触发搜索（防抖）
- 搜索范围：便签标题 + 正文纯文本内容
- 搜索方式：不区分大小写的子串匹配

**搜索结果展示**：
- 匹配的便签正常显示
- 不匹配的便签隐藏
- 如果无匹配结果，显示"无匹配便签"提示
- 搜索框有内容时，在搜索框右侧显示匹配数量 "(3/10)"

**清空操作**：
- 点击 ✕ 按钮：清空搜索框，恢复显示所有便签
- 按 Esc 键：同上
- 关闭列表面板：保留搜索内容（下次打开时仍生效）

**边界情况处理**：
1. **搜索时切换便签**：允许，点击搜索结果中的便签后，列表收起，搜索框保留内容
2. **搜索时新建便签**：新建后列表收起，搜索框清空
3. **搜索时删除便签**：删除后重新过滤，如果当前便签被删除且有其他便签，自动加载第一个便签
4. **空列表搜索**：搜索框禁用，显示灰色

### 2.3 技术实现要点

**数据过滤**：
```python
def _filter_notes(self, query: str) -> list[Note]:
    if not query.strip():
        return self._svc.get_all()
    
    query_lower = query.lower()
    all_notes = self._svc.get_all()
    filtered = []
    
    for note in all_notes:
        # 搜索标题
        if note.title and query_lower in note.title.lower():
            filtered.append(note)
            continue
        
        # 搜索正文（去除HTML标签）
        plain_text = self._extract_plain_text(note.body_html)
        if query_lower in plain_text.lower():
            filtered.append(note)
    
    return filtered
```

**防抖处理**：
- 使用 QTimer.singleShot(300, callback) 实现输入防抖
- 避免每次按键都触发过滤，提升性能

**状态管理**：
- 搜索关键词存储在 `self._search_query` 中
- 列表刷新时始终应用当前搜索过滤

## 三、功能2：独立便签窗口置顶按钮

### 3.1 需求分析

**核心需求**：
- 每个独立便签窗口可以单独控制是否置顶
- 置顶状态独立于主窗口的置顶状态
- 置顶按钮应清晰可见且易于操作

**使用场景**：
- 用户拖出多个便签，希望某个重要便签始终在最前
- 主窗口不置顶，但某个便签需要置顶
- 临时参考某个便签内容时，置顶该便签窗口

### 3.2 设计方案

#### 3.2.1 UI布局

**位置选择**：在独立便签窗口的标题栏右侧，关闭按钮左边

**布局结构**：
```
┌─────────────────────────────────────┐
│ 便签        [📌]  [✕]               │ ← 标题栏（34px高）
├─────────────────────────────────────┤
│ 标题输入框                          │
├─────────────────────────────────────┤
│                                     │
│ 正文编辑区                          │
│                                     │
│                                     │
│                                  ‹  │ ← 格式工具栏（收起状态）
└─────────────────────────────────────┘
```

**置顶按钮设计**：
- 图标：📌（图钉）
- 尺寸：22x22px
- 状态：
  - 未置顶：半透明灰色图钉，悬停时高亮
  - 已置顶：实心彩色图钉（使用主题强调色）
- 位置：关闭按钮左侧，间隔4px
- Tooltip：
  - 未置顶时："窗口置顶"
  - 已置顶时："取消置顶"

#### 3.2.2 交互逻辑

**置顶切换**：
- 点击按钮：切换置顶状态
- 置顶时：窗口 flags 添加 `Qt.WindowStaysOnTopHint`
- 取消置顶时：窗口 flags 移除 `Qt.WindowStaysOnTopHint`

**状态持久化**：
- **方案A（推荐）**：不持久化，每次打开默认不置顶
  - 理由：独立便签窗口是临时性的，关闭后状态丢失符合直觉
  - 优点：实现简单，无需额外存储
  
- **方案B**：持久化到数据库
  - 在 Note 模型中添加 `pinned` 字段
  - 每次打开该便签的独立窗口时，恢复置顶状态
  - 缺点：增加复杂度，且可能导致用户困惑（为什么这个窗口总是置顶？）

**推荐采用方案A**。

**视觉反馈**：
- 点击时按钮有轻微缩放动画（可选）
- 置顶/取消置顶时，按钮图标颜色立即变化
- 无需额外的提示信息（图标状态已足够清晰）

#### 3.2.3 边界情况处理

1. **多个独立窗口同时置顶**：允许，各窗口独立控制
2. **主窗口置顶 vs 独立窗口置顶**：互不影响，各自独立
3. **拖出时的默认状态**：新拖出的窗口默认不置顶
4. **置顶窗口关闭后重新拖出**：重新拖出时不置顶（采用方案A）

### 3.3 技术实现要点

**窗口 flags 动态修改**：
```python
def _toggle_pin(self):
    self._is_pinned = not self._is_pinned
    
    # 保存当前位置和大小
    geo = self.geometry()
    
    # 修改窗口 flags
    if self._is_pinned:
        self.setWindowFlags(
            Qt.FramelessWindowHint | Qt.WindowStaysOnTopHint | Qt.Tool
        )
    else:
        self.setWindowFlags(
            Qt.FramelessWindowHint | Qt.Tool
        )
    
    # 恢复位置和大小（修改 flags 会导致窗口隐藏）
    self.setGeometry(geo)
    self.show()
    
    # 更新按钮样式
    self._update_pin_button()
```

**图钉图标绘制**：
- 使用 QPainter 绘制图钉图标（参考主窗口的图钉按钮实现）
- 或使用 Unicode 字符：📌 (U+1F4CC)
- 颜色：
  - 未置顶：`theme.text_secondary` + 50% 透明度
  - 已置顶：`theme.priority_high` 或主题强调色

## 五、潜在问题与解决方案

### 5.1 便签列表搜索相关问题

**问题1：搜索性能**
- **场景**：便签数量达到100+时，每次输入都过滤可能卡顿
- **解决**：
  - 使用防抖（300ms）减少过滤频率
  - 正文搜索时，只搜索前500字符（大部分情况足够）
  - 如果仍有性能问题，考虑后台线程过滤

**问题2：搜索框占用空间**
- **场景**：列表高度本就有限，搜索框占用28px可能显得拥挤
- **解决**：
  - 搜索框设计为可收起（默认展开）
  - 或：只在列表高度 > 200px 时显示搜索框
  - **推荐**：始终显示，因为搜索是高频需求

**问题3：搜索状态混乱**
- **场景**：用户搜索后关闭列表，再打开时忘记之前有搜索，看到的列表不完整
- **解决**：
  - 方案A：关闭列表时自动清空搜索（推荐）
  - 方案B：保留搜索，在搜索框中明显提示匹配数量 "(搜索中: 3/10)"

**问题4：搜索时删除便签**
- **场景**：搜索结果中删除便签后，如果结果为空，用户可能困惑
- **解决**：
  - 删除后重新过滤
  - 如果结果为空，自动清空搜索框并显示所有便签
  - 提示："已删除，搜索已清空"

### 5.2 独立窗口置顶相关问题

**问题1：修改窗口 flags 导致窗口闪烁**
- **场景**：调用 `setWindowFlags()` 会导致窗口短暂隐藏再显示
- **解决**：
  - 保存当前几何位置
  - 修改 flags 后立即恢复位置
  - 使用 `show()` 而非 `setVisible(True)`
  - 代码示例见 3.3 节

**问题2：置顶按钮与关闭按钮视觉区分**
- **场景**：两个按钮靠得太近，用户可能误点
- **解决**：
  - 保持4px间距
  - 置顶按钮使用不同颜色（灰色 vs 红色关闭按钮）
  - 悬停时有明显的视觉反馈

**问题3：置顶状态不直观**
- **场景**：用户不确定窗口是否已置顶
- **解决**：
  - 图钉图标颜色差异明显（灰色 vs 红色）
  - Tooltip 清晰说明当前状态和点击后的效果
  - 可选：置顶时标题栏背景色略微变化

**问题4：多窗口置顶冲突**
- **场景**：多个独立窗口都置顶，用户可能分不清哪个在最前
- **解决**：
  - 这是预期行为，多个置顶窗口可以共存
  - 最后点击的窗口会在最前（Qt 默认行为）
  - 无需特殊处理

**问题5：置顶窗口与主窗口的关系**
- **场景**：主窗口置顶，独立窗口不置顶，可能被主窗口遮挡
- **解决**：
  - 独立窗口默认使用 `Qt.Tool` flag，会在主窗口之上
  - 即使不置顶，也不会被主窗口遮挡
  - 只有其他应用的窗口可能遮挡未置顶的独立窗口

## 八、总结

### 8.1 设计亮点

1. **搜索功能**：
   - 位置合理，不影响现有布局
   - 实时过滤，响应迅速
   - 支持标题和正文搜索，覆盖全面

2. **置顶功能**：
   - 独立控制，互不干扰
   - 视觉反馈清晰
   - 实现简单，无需持久化

### 8.2 风险评估

**低风险**：
- 置顶功能实现简单，风险低
- 搜索功能逻辑清晰，边界情况已考虑

**中风险**：
- 搜索性能（便签数量多时）- 已有防抖和优化方案
- 窗口 flags 修改可能闪烁 - 已有解决方案

**建议**：
- 先实现基础功能，测试后再优化细节
- 搜索功能可以先只搜索标题，后续再加正文搜索
- 置顶按钮图标可以先用 Unicode 字符，后续再用 QPainter 绘制

### 8.3 用户体验考虑

1. **搜索功能**：
   - 防抖避免卡顿
