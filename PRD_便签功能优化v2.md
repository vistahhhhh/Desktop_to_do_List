# 桌面待办 — 便签功能优化 PRD v2.0

> 版本: v2.0 | 日期: 2026-03-27 | 状态: 设计中

---

## 一、优化概述

基于现有便签功能（v1.0），进行三项UI/UX优化，提升便签使用体验的轻量感和灵活性。

### 核心优化点

1. **悬浮列表设计** - 便签列表改为悬浮卡片，不占满整个页面
2. **底部栏透明化** - 去除底部栏背景色，只保留按钮视觉
3. **便签拖出功能** - 支持将单个便签拖出为独立悬浮小窗口

---

## 二、优化详情

### 2.1 悬浮列表设计

#### 现状问题
- 当前便签列表从底部上拉，占据整个编辑区域
- 视觉上过于厚重，遮挡了正在编辑的内容
- 列表宽度与窗口同宽，缺乏层次感

#### 优化方案

**视觉设计：**
```
┌─────────────────────────────────────────────┐
│ 📌  桌面待办  便签          ⚙  —  ×         │
├─────────────────────────────────────────────┤
│  ‹                                          │  ← 格式工具栏（右侧）
│                                             │
│  标题…                                      │
│                                             │
│  正文内容…                                  │
│                                             │
│                                             │
│                    ┌──────────────────┐    │  ← 悬浮列表卡片
│                    │ ▼ 便签列表        │    │
│                    ├──────────────────┤    │
│                    │ ● 开会笔记  03-26│    │
│                    │ ● 今日感悟  03-25│    │
│                    │ ● 项目灵感  03-24│    │
│                    └──────────────────┘    │
├─────────────────────────────────────────────┤
│  ＋           ▲  开会笔记…              🗑   │
└─────────────────────────────────────────────┘
```

**关键特性：**
- **悬浮定位**：列表卡片浮在编辑区上方，不推开内容
- **宽度对齐**：与底部"展开便签列表"按钮（中间按钮）同宽
- **位置锚定**：列表底部对齐底部栏顶部，向上展开
- **半透明背景**：列表卡片带有轻微透明度和阴影，增强层次感
- **圆角设计**：与应用整体风格保持一致

**交互行为：**
- 点击底部中间按钮 → 列表从底部向上弹出（动画：淡入 + 向上滑动）
- 点击列表外任意区域 → 列表收起（动画：淡出 + 向下滑动）
- 点击列表标题栏"▼ 便签列表" → 列表收起
- 列表内支持滚轮滚动浏览

### 2.3 便签拖出功能

#### 功能说明
用户可以将任意便签拖出主窗口，形成一个独立的悬浮小窗口，专注查看或编辑该便签。

#### 使用场景
- **多任务参考**：拖出一个便签作为参考资料，同时在主窗口编辑待办
- **会议记录**：拖出会议笔记便签，置顶在屏幕一角，边开会边记录
- **灵感暂存**：快速拖出一个便签记录想法，不影响主窗口当前工作流

#### 交互设计

**拖出操作：**
```
1. 鼠标按住便签标题输入框（或标题栏区域）
2. 拖动鼠标超过一定距离（如 20px）
3. 触发拖出效果：
   - 主窗口便签内容保持不变
   - 鼠标位置生成一个半透明的便签预览
4. 松开鼠标 → 在松开位置创建独立便签窗口
```

**独立便签窗口设计：**
```
┌─────────────────────────────────────┐
│  便签                          ‹  ✕ │  ← 标题栏（可拖动）
├─────────────────────────────────────┤
│  开会笔记                           │  ← 标题（只读或可编辑）
│                                     │
│  1. 讨论项目进度                    │  ← 正文内容
│  2. 确定下周里程碑                  │
│  3. 分配任务给团队成员              │
│                                     │
│                                     │
├─────────────────────────────────────┤
│                                  ‹  │  ← 底部工具栏（只有格式按钮）
└─────────────────────────────────────┘
```

**窗口特性：**
- **尺寸**：默认 300x400px，可调整大小（拖拽边缘）
- **置顶**：默认始终置顶（可通过设置取消）
- **无边框**：圆角无边框设计，与主窗口风格一致
- **半透明**：继承主窗口透明度设置
- **可拖动**：按住标题栏拖动窗口位置
- **可缩放**：拖拽窗口边缘调整大小

**底部工具栏：**
- **右侧格式面板**：保留 B/I/U 格式按钮（展开/收起）
- **无其他按钮**：不显示"＋"、"列表"、"🗑"按钮
- **简化设计**：只保留必要的编辑功能

**数据同步：**
- **实时同步**：独立窗口与主窗口共享同一便签数据
- **自动保存**：编辑后 1 秒自动保存，双向同步
- **关闭行为**：
  - 点击 ✕ → 关闭独立窗口，数据保存
  - 主窗口切换到该便签 → 独立窗口不受影响（可同时编辑）
  - 主窗口删除该便签 → 独立窗口自动关闭并提示

**拖出触发区域：**
- **标题输入框**：按住标题输入框拖动
- **视觉反馈**：
  - 鼠标悬停标题框时，显示提示："按住拖动可拖出便签"
  - 开始拖动时，鼠标变为抓手图标
  - 拖动过程中，显示半透明预览窗口跟随鼠标

## 四、技术实现要点

### 4.1 悬浮列表

**实现方式：**
- 使用 `QFrame` 作为列表容器，设置为主窗口的子控件
- 通过绝对定位（`move()` + `resize()`）实现悬浮效果
- 监听底部按钮的几何变化，动态调整列表位置和宽度

**关键代码结构：**
```python
class NotePanel(QWidget):
    def _build_list_panel(self):
        # 列表面板不再嵌入 bottom_container
        # 而是作为 NotePanel 的直接子控件
        panel = QFrame(self)
        panel.setObjectName("NoteListFloating")  # 新样式
        panel.setWindowFlags(Qt.Popup)  # 或使用绝对定位
        # ...
        return panel
    
    def _position_list_panel(self):
        # 根据 current_btn 的位置和大小定位列表
        btn_geo = self.current_btn.geometry()
        btn_global = self.current_btn.mapTo(self, btn_geo.topLeft())
        
        panel_width = btn_geo.width()
        panel_height = 160
        panel_x = btn_global.x()
        panel_y = btn_global.y() - panel_height - 8
        
        self._list_panel.setGeometry(panel_x, panel_y, panel_width, panel_height)
```

**样式定义：**
```python

QFrame#NoteListFloating {
    background-color: rgba(..., 0.98);  # 高不透明度
    border: 1px solid ...;
    border-radius: 8px;
    # 阴影通过 QGraphicsDropShadowEffect 实现
}
```

### 4.3 便签拖出功能

**核心组件：**
1. **FloatingNoteWindow** - 独立便签窗口类
2. **拖动检测** - 在 `NotePanel.title_edit` 上监听鼠标事件
3. **数据共享** - 独立窗口与主窗口共享 `Note` 对象引用

**实现流程：**
```python
class NotePanel(QWidget):
    def __init__(self, ...):
        # ...
        self._drag_start_pos = None
        self._is_dragging = False
        self.title_edit.installEventFilter(self)
    
    def eventFilter(self, obj, event):
        if obj == self.title_edit:
            if event.type() == QEvent.MouseButtonPress:
                if event.button() == Qt.LeftButton:
                    self._drag_start_pos = event.globalPos()
            elif event.type() == QEvent.MouseMove:
                if self._drag_start_pos:
                    delta = event.globalPos() - self._drag_start_pos
                    if delta.manhattanLength() > 20:
                        self._start_detach()
                        self._drag_start_pos = None
            elif event.type() == QEvent.MouseButtonRelease:
                self._drag_start_pos = None
        return super().eventFilter(obj, event)
    
    def _start_detach(self):
        """拖出便签为独立窗口"""
        if not self._current_note:
            return
        floating_win = FloatingNoteWindow(
            self._svc, 
            self._current_note.id,
            parent=None
        )
        floating_win.show()


class FloatingNoteWindow(QWidget):
    """独立便签悬浮窗口"""
    def __init__(self, note_service, note_id, parent=None):

        self._note_id = note_id

        # 无边框、置顶、半透明
        self.setWindowFlags(Qt.FramelessWindowHint | Qt.WindowStaysOnTopHint | Qt.Tool)
        self.setAttribute(Qt.WA_TranslucentBackground)
        self.resize(300, 400)
        
        self._setup_ui()
        self._load_note()
```

## 五、用户体验流程

### 5.1 悬浮列表使用流程
```
1. 用户点击底部"▲ 开会笔记…"按钮
   → 列表从底部向上弹出（动画：200ms 淡入 + 上移）
   → 按钮文字变为"▼ 开会笔记…"

2. 用户滚轮浏览历史便签
   → 列表内容滚动，列表位置不变

3. 用户点击某条便签
   → 列表收起（动画：200ms 淡出 + 下移）
   → 编辑区加载所选便签
   → 按钮文字更新为"▲ [新便签名]"

4. 用户点击列表外区域（或标题栏）
   → 列表收起，不切换便签
```

### 5.2 便签拖出使用流程
```
1. 用户鼠标悬停在标题输入框
   → 显示提示："按住拖动可拖出便签"

2. 用户按住鼠标左键并拖动
   → 鼠标变为抓手图标
   → 生成半透明预览窗口跟随鼠标

3. 用户拖动到目标位置后松开鼠标
   → 预览窗口消失
   → 在松开位置创建独立便签窗口
   → 主窗口保持不变

4. 用户在独立窗口编辑便签
   → 自动保存（1秒延迟）
   → 主窗口如果打开同一便签，内容同步更新

5. 用户关闭独立窗口
   → 窗口消失，数据已保存
   → 主窗口不受影响
```
