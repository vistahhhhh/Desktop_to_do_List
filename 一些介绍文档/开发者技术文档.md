## 3. 架构设计

### 3.1 三层架构

```
┌─────────────────────────────────────────┐
│                 UI 层                    │
│  main_window / tag_sidebar / task_list  │
│  task_item / task_editor / settings     │
├─────────────────────────────────────────┤
│              Service 层                  │
│  TaskService / TagService / FilterService│
│  NoteService                             │
├─────────────────────────────────────────┤
│              Model 层                    │
│  Task / Tag / Note / database / ConfigManager  │
└─────────────────────────────────────────┘
```

- **Model 层**：SQLAlchemy ORM 模型 + 数据库初始化/迁移 + JSON 配置管理
- **Service 层**：纯业务逻辑，不依赖 UI。所有数据操作通过 Service 层完成
- **UI 层**：PyQt5 控件，通过 Qt 信号/槽机制与 Service 层交互

### 3.2 数据流

```
用户操作 → UI控件信号 → MainWindow槽函数 → Service方法 → SQLAlchemy ORM → SQLite
                                              ↓
                                     MainWindow._refresh_tasks()
                                              ↓
                                     UI控件更新显示
```

## 4. 关键模块详解

### 4.1 数据库与模型

#### `src/models/database.py` — 数据库初始化

- **`init_db()`**：调用 `Base.metadata.create_all()` 创建表，然后执行 `_migrate()` 轻量迁移
- **`_migrate(engine)`**：通过 `sqlalchemy.inspect` 检测现有列，用 `ALTER TABLE ADD COLUMN` 补充缺失字段（`is_deleted`、`icon`、`task_date`）。确保老版本数据库也能无缝升级
- **`get_session()`**：单例会话工厂，`expire_on_commit=False` 避免访问属性时触发额外查询

#### `src/models/task.py` — Task 模型

```python
class Task(Base):
    __tablename__ = "tasks"
    id          # 主键自增
    title       # 标题（必填，200字以内）
    description # 详细描述（可选）
    task_type   # 'short_term' | 'weekly' | 'long_term'
    due_date    # 截止日期（长期/周计划使用）
    priority    # 'high' | 'medium' | 'low'
    status      # 'todo' | 'in_progress' | 'done' | 'cancelled'
    is_deleted  # 0=正常, 1=回收站（软删除）
    task_date   # 当日任务所属日期（short_term 使用）
    tags        # 多对多关系 → Tag（通过 task_tags 关联表）
```

- **多对多关系**：`task_tags` 关联表，`lazy="joined"` 预加载标签避免 N+1 查询
- **软删除**：`is_deleted` 字段实现回收站功能，所有查询默认过滤 `is_deleted != 1`

#### `src/models/note.py` — Note 模型（便签）

- 基础字段：`title`、`body_html`、`created_at`、`updated_at`、`deleted`
- `display_name()`：
  - 有标题：返回标题（压缩空白，确保单行）
  - 无标题：从 `body_html` 提取纯文本前 10 字，供便签列表显示

### 4.2 业务服务层

#### `src/services/task_service.py` — TaskService

| 方法 | 功能 |
|------|------|
| `create_task()` | 创建任务，自动设置 `task_date=today`（短期任务） |
| `update_task()` | 通用更新，支持所有字段 + 标签关系变更 |
| `update_status()` | 状态切换快捷方法 |
| `delete_task()` | 软删除（`is_deleted=1`） |
| `restore_task()` | 从回收站恢复（`is_deleted=0`） |
| `permanent_delete_task()` | 物理删除（`session.delete`） |
| `carry_forward()` | 遗留任务移交今日（更新 `task_date`） |
| `get_history_tasks(max_days=10)` | 获取最近10天的已过去短期任务，按日期分组，最新在前 |
| `clear_history_tasks()` | 永久清空所有历史短期任务 |

#### `src/services/filter_service.py` — FilterService（智能清单）

| 方法 | 筛选逻辑 |
|------|----------|
| `get_today_tasks()` | `task_date==今天` 的短期任务 + 过去遗留未完成任务（`task_date<今天 且 status=todo/in_progress`） |
| `get_week_tasks()` | `task_type=='weekly'` 且未删除，按截止日期排序 |
| `get_long_term_tasks()` | `task_type=='long_term'` 且未删除 |
| `get_tasks_by_tag()` | JOIN `task_tags` 按标签 ID 筛选 |

- **排序规则**：所有清单使用 `case` 表达式将已完成任务排到末尾：`case((status=='done', 1), else_=0)`

#### `src/services/note_service.py` — NoteService（便签服务）

| 方法 | 功能 |
|------|------|
| `get_all()` | 获取所有未删除便签（按 `updated_at` 倒序） |
| `create()` | 创建空便签（落库生成 ID） |
| `save(note, title, body_html)` | 保存便签内容 |
| `delete(note_id)` | 软删除便签（回收站） |
| `restore(note_id)` | 从回收站恢复 |
| `permanent_delete(note_id)` | 永久删除 |
| `clear_deleted()` | 清空回收站 |

### 4.3 UI 层

#### `src/ui/main_window.py` — MainWindow（核心控制器，1146行）

这是整个应用的中枢，职责包括：

**窗口管理**
- 无边框 + 透明背景 + 置顶：`Qt.FramelessWindowHint | Qt.WindowStaysOnTopHint | Qt.Tool`
- 边缘缩放：`eventFilter` 全局事件过滤器检测鼠标位置（6px 边缘/16px 角落），动态切换光标，拖拽时调用 `_do_resize()`
- 标题栏拖拽：`eventFilter` 中检测 `local.y() < 48` 且非按钮点击，启动拖拽移动
- 位置记忆：`_save_position()` / `_load_position()` 通过 ConfigManager 持久化

**主要方法**

| 方法 | 说明 |
|------|------|
| `_setup_ui()` | 构建整个 UI 树：标题栏 → 内容区（侧栏+任务列表）→ 底部按钮 |
| `_refresh_tasks()` | 根据 `sidebar.get_current_key()` 调用对应 FilterService 方法并刷新列表 |
| `_on_filter_changed()` | 侧栏点击信号处理：更新标题文字 + 副标题显示 + 刷新任务 |
| `_on_add_task()` | 打开 TaskEditorDialog 新建任务，自动推断任务类型和默认标签 |
| `_on_task_edit()` | 双击任务 → 打开编辑弹窗 → 保存后刷新 |
| `_on_open_trash()` | 构建回收站弹窗：滚动列表 + 恢复/删除/清空 |
| `_on_open_history()` | 构建待办回顾弹窗：按日期分组 + 状态标记 + 清空 |
| `_show_confirm()` | 通用主题化确认弹窗（返回 bool） |
| `_show_tag_dialog()` | 标签创建/编辑弹窗（图标网格选择） |
| `switch_theme()` | 切换主题 → 重新生成 QSS → 重建阴影效果 |
| `bring_to_front()` | 单实例唤醒：`show()` + `showNormal()` + `SetForegroundWindow(hwnd)` |

#### `src/ui/note_panel.py` — NotePanel（便签面板）

主要能力：
- 便签列表（支持展开/收起）
- 便签编辑器（标题 + 富文本正文）
- 搜索过滤（实时过滤 + 显示匹配数量）
- 便签回收站（恢复/永久删除/清空）
- 自动保存策略：
  - 当内容为空时不创建便签
  - 当已有便签被清空为无内容时，永久删除并回到“未创建状态”

对外接口：
- `get_current_note_id()`：拖出便签时获取当前便签 ID
- `ensure_note_for_detach()`：即便当前是空白，也创建一个临时便签用于拖出
- `start_new_note()`：拖出后主面板切换到新空白
- `load_note_by_id(note_id)`：浮动便签关闭后回显该便签（若已被删除返回 False）

#### `src/ui/floating_note.py` — FloatingNoteWindow（浮动便签）

- 从主窗口拖出后创建独立窗口展示单条便签
- 独立置顶按钮：`set_pinned()` / `_toggle_pin()` 通过 `Qt.WindowStaysOnTopHint` 切换
- 自动保存策略：无内容时永久删除，确保不保存空便签
- 关闭信号：关闭时向主窗口发送便签 ID（若该便签未被“空内容删除”）

#### `src/ui/tag_sidebar.py` — TagSidebar（左侧标签栏）

- **三区布局**：顶部固定智能清单（3个按钮）→ 中间可滚动自定义标签区 → 底部待办回顾按钮
- **TagButton**：继承 QPushButton，`set_active()` 切换 `TagBtn`/`TagBtnActive` objectName 实现激活样式
- **右键菜单**：`contextMenuEvent` 检测点击目标——自定义标签按钮则显示编辑/删除，空白区域则显示新建
- **tooltip**：标签区域悬停显示"右键新建标签"

#### `src/ui/task_item.py` — TaskItemWidget（任务卡片）

- **三种显示模式**：
  - 正常任务：状态圆圈 + 标题 + 日期 + 优先级标记
  - 遗留任务（`is_carryover`）：斜体标题 + "移交今日"按钮
  - 周计划过期（`is_week_overdue`）：斜体标题 + "重新编辑日期"按钮
- **状态流转**：`STATUS_FLOW` 字典定义 `todo↔done`，点击圆圈一键切换
- **拖拽排序**：`mouseMoveEvent` 检测移动距离 >20px 触发 `QDrag`，生成半透明拖影

#### `src/ui/task_editor.py` — TaskEditorDialog（编辑弹窗）

- **智能默认值**：根据当前清单自动设置任务类型（今日→短期，本周→周计划，长期→长期，标签→长期+预选标签）
- **类型联动**：`_on_type_changed()` 控制周几选择/日期选择器的显隐
- **本周计划日期**：`_populate_weekday_combo()` 填充从今天起未来10天的 ComboBox
- **标签选择器**：`TagChip` 组件，点击切换选中状态，样式动态更新
- **键盘快捷键**：Enter 保存（描述框内 Enter 正常换行）
- **弹窗可拖拽**：自定义 `mousePressEvent`/`mouseMoveEvent`

## 5. 单实例运行与前台唤醒

### 实现方案：QLocalServer / QLocalSocket

**`main.py` 启动流程**：

```
启动 → QApplication → _try_send_show() 尝试连接已有实例
  ├── 连接成功 → 发送 "show" → 退出（已有实例会收到唤醒信号）
  └── 连接失败 → 创建 QLocalServer 监听 → 启动 MainWindow
                      ↓
                  newConnection 信号
                      ↓
                  window.bring_to_front()
```

**`bring_to_front()` 实现**：
```python
self.show()           # 从隐藏状态恢复
self.showNormal()     # 从最小化恢复
self.raise_()         # 提升窗口层级
self.activateWindow() # 激活窗口
# Windows API 强制前台（绕过系统限制）
ctypes.windll.user32.SetForegroundWindow(int(self.winId()))
```

**为什么不用 Windows 消息广播**：之前尝试过 `RegisterWindowMessage` + `PostMessage(HWND_BROADCAST)` 方案，但因为窗口使用 `Qt.Tool` 标志（不出现在任务栏），且 `hide()` 后原生窗口不活跃，导致无法收到广播消息。QLocalServer 通过命名管道通信，完全不依赖窗口状态。

## 7. 打包与发布

### 7.1 PyInstaller（`build.spec`）

- **模式**：`onedir`（目录模式），比 `onefile` 启动快很多（无需解压到临时目录）
- **优化**：`optimize=2`（字节码优化）
- **排除模块**：大量未使用的 PyQt5 子模块和标准库模块，减小体积
- **保留**：`PyQt5.QtNetwork`（QLocalServer 需要）
- **图标**：`icon='app图标.ico'`，exe 文件自带图标
- **路径**：`pathex=[str(Path.cwd())]`，确保 `src/` 包可被稳定分析
- **hiddenimports**：补充便签相关模块（避免漏收集导致安装版缺少便签功能）

**构建命令**：
```bash
pyinstaller -y build.spec
```

### 7.2 Inno Setup（`installer.iss`）

- 复制整个 `dist\桌面待办\` 目录到安装目标
- 额外复制 `app图标.ico` 到安装根目录（桌面快捷方式图标指向它）
- `SetupIconFile=app图标.ico`：安装程序自身的图标
- `PrivilegesRequired=lowest`：无需管理员权限
- `Compression=lzma2/ultra64` + `SolidCompression=yes`：提高安装包压缩率
- 自动创建桌面快捷方式 + 开始菜单项

**编译命令**：
```bash
"C:\...\Inno Setup 6\ISCC.exe" installer.iss
```

### 7.3 图标处理

源文件 `app图标.png`（透明 PNG）转换为 `app图标.ico`：
```python
from PIL import Image
img = Image.open('app图标.png')
img.save('app图标.ico', format='ICO',
         sizes=[(256,256),(128,128),(64,64),(48,48),(32,32),(16,16)])
```

## 9. 已解决的关键问题

| 问题 | 根因 | 解决方案 |
|------|------|---------|
| 安装版创建任务报 `readonly database` | 安装目录（`Program Files`）默认只读 | 用户数据迁移到 `%LOCALAPPDATA%\桌面待办\` |
| exe 启动极慢（10+ 秒） | PyInstaller `onefile` 模式每次启动需解压到临时目录 | 改用 `onedir` 模式 + 排除未使用模块 |
| 桌面快捷方式无图标 | Inno Setup 指向 `{app}\app图标.ico` 但 ico 在 `_internal/` 子目录 | 在 installer.iss 中单独复制 ico 到 `{app}` 根 |
| QSS 重刷后阴影消失 | `setStyleSheet()` 会使 `QGraphicsDropShadowEffect` 失效 | 每次 `_apply_theme()` 后调用 `_rebuild_shadow()` 重建 |
| ConfigManager 浅拷贝导致默认值被修改 | `_merge_defaults` 直接引用 `DEFAULT_CONFIG` 字典 | 改用 `copy.deepcopy` |
| 单实例广播消息无效 | `Qt.Tool` 窗口 + `hide()` 后不接收 Windows 广播消息 | 改用 `QLocalServer`/`QLocalSocket` 命名管道方案 |
| 安装包运行缺少便签功能 | PyInstaller 分析路径/隐式导入不完整，导致模块漏收集 | 在 `build.spec` 中补齐 `pathex` 与便签相关 `hiddenimports`，重新构建 dist 再编译 setup |
