# 桌面待办 — 便签功能 PRD

> 版本: v1.0 | 日期: 2026-03-26 | 状态: 设计中

---

## 一、功能概述

在"桌面待办"应用中集成**轻量便签**功能，用户可以随时记录碎片化想法、参考资料、会议笔记等非结构化内容。

### 核心理念

```
便签 = 随时可写的草稿纸
待办 = 必须完成的行动项
```

---

## 二、用户场景

| 场景 | 描述 |
|------|------|
| **快速记录** | 开会时随手记下要点，不想打断当前工作流 |
| **任务备注** | 某个任务需要详细的参考信息（链接、步骤、注意事项） |
| **灵感暂存** | 突然想到一个点子，先记下来，之后再转为待办 |
| **日记/复盘** | 每天简短记录今日感悟，回顾时查看 |

## 四、功能详细说明

### 4.1 顶部格式工具栏

位于共用顶栏正下方，横排三个按钮：

| 按钮 | 功能 | 快捷键（可选） |
|------|------|----------------|
| **B** | 加粗选中文字 | Ctrl+B |
| *I* | 斜体选中文字 | Ctrl+I |
| <u>U</u> | 下划线选中文字 | Ctrl+U |

- 选中文字后点击对应按钮即可切换格式，再次点击取消
- 按钮在对应格式激活时呈**高亮/按下**视觉状态


### 4.2 便签编辑区

| 区域 | 说明 |
|------|------|
| **标题栏** | 单行输入，placeholder 为"标题（可选）" |
| **正文区** | 多行富文本，支持 B/I/U 格式，placeholder 为"开始记录…" |
| **自动保存** | 用户停止输入 **1 秒**后自动保存，无需手动操作；应用关闭时也强制保存 |

### 4.3 底部操作栏

```
┌─────────────────────────────────────────────┐
│  ＋          ▲  开会笔记 2026-03-26      🗑  │
└─────────────────────────────────────────────┘
```

| 区域 | 元素 | 行为 |
|------|------|------|
| 左侧 | **＋** 按钮 | 创建新便签，编辑区清空，聚焦标题 |
| 中间 | **便签列表入口** | 显示当前便签标题（无标题则显示正文前7字+"…"）；点击展开上拉列表 |
| 右侧 | **🗑 回收站** | 删除当前便签，弹出二次确认 |

### 4.4 便签列表（上拉面板）

点击底部中间区域后，从底部滑出一个**上拉浮层**：

```
┌─────────────────────────────────────────────┐
│ 📌  桌面待办  便签          ⚙  —  ×         │
├─────────────────────────────────────────────┤
│  B  I  U                                    │
├─────────────────────────────────────────────┤
│                                             │

│                                             │
│  正文内容…                                  │
│                                             │
├╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌┤
│  ▼  便签列表                                │  ← 列表面板标题（点击收起）
├─────────────────────────────────────────────┤

**列表面板行为：**
- 从底部向上滑出，遮盖部分编辑区（不全屏覆盖）

- 点击某条便签 → 关闭面板 + 加载该便签到编辑区
- 每条右侧 **🗑** 图标 → 删除该条（弹出确认提示）
- 点击面板标题或编辑区 → 收起面板

## 五、数据结构

### Note 模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增主键 |
| `title` | TEXT | 标题，可为空 |
| `body_html` | TEXT | 富文本正文（HTML 格式，保留 B/I/U 标记） |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 最后修改时间 |
| `deleted` | BOOLEAN | 软删除标记（回收站逻辑） |

### 列表显示名规则

```python
display_name = note.title.strip() if note.title.strip() \
               else (plain_text[:7] + "…" if len(plain_text) > 7 else plain_text)
```

## 六、交互流程

### 6.1 新建便签

```
点击 ＋
  → 编辑区清空
  → 聚焦标题输入框
  → 1 秒无输入后自动创建新记录并保存
```

### 6.2 切换便签

```
打开列表面板
  → 点击某条便签
  → 当前便签立即保存
  → 加载所选便签内容到编辑区
  → 面板收起
```

### 6.3 删除便签

```
点击底栏 🗑（删当前）或列表内 🗑（删指定）
  → 弹出确认："确认删除这条便签？"
  → 确认 → 软删除（deleted=True），列表中消失
  → 若删除的是当前正在编辑的便签 → 自动跳到最新一条；若无便签则新建空白
```

### 6.4 格式应用

```
选中文字 → 点击 B / I / U
  → 对选区切换对应 HTML 标签（<b>/<i>/<u>）
  → 按钮视觉状态同步高亮
  → 1 秒后自动保存
```

## 八、非功能要求

| 项目 | 要求 |
|------|------|
| **自动保存延迟** | 停止输入后 ≤ 1 秒完成写库 |
| **列表加载** | 列表面板打开时 ≤ 100ms 渲染完毕 |
| **主题适配** | 编辑区、工具栏、列表面板均遵循当前主题配色（深色/浅色） |
| **字体一致** | 使用与待办主界面相同的字体和字号设置 |
| **数据存储** | 存入同一 SQLite 数据库，新增 `notes` 表 |
