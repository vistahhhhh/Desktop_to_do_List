**问题分析：**
- ❌ 列表从底部上拉，占据整个编辑区域，遮挡正在编辑的内容

- ❌ 缺乏层次感，像是页面切换而非弹出菜单

```

**关键设计要点：**

1. **定位锚点**
   ```
   列表左边缘 = 中间按钮左边缘
   列表右边缘 = 中间按钮右边缘
   列表底部 = 底部栏顶部 - 8px（间隙）
   列表高度 = 160px（固定，内部可滚动）
   ```

2. **视觉层次**
   ```
   层级1（最底）：编辑区背景
   层级2：正文内容（被列表遮挡部分变暗）
   层级3（最顶）：悬浮列表卡片 + 阴影
   ```

3. **列表卡片样式**——需要根据不同主题颜色自行设定！！！！以下颜色仅供参考
   - 背景色：`card_bg` + 98% 不透明度
   - 边框：1px solid `border_color`
   - 圆角：8px
   - 阴影：模糊16px，偏移(0, 4px)，颜色 rgba(0,0,0,0.15)

## 优化点3：便签拖出功能

### 功能概述

将便签从主窗口拖出，形成独立的悬浮小窗口，方便多任务参考。

---

**拖动检测逻辑：**
```
1. 鼠标按下（MouseButtonPress）→ 记录起始位置
2. 鼠标移动（MouseMove）→ 计算移动距离
3. 距离 > 20px → 触发拖出
4. 生成半透明预览窗口跟随鼠标
5. 鼠标松开（MouseButtonRelease）→ 在松开位置创建独立窗口
```

**收起状态（默认）：**
```
┌─────────────────────────────────────────┐

│                                         │
├─────────────────────────────────────────┤
│                                      ‹  │ ← 只显示小箭头（右侧）

**展开状态（点击箭头后）：**
```
┌─────────────────────────────────────────┐

│                                         │
├─────────────────────────────────────────┤
│                          [B] [I] [U]  › │ ← B/I/U按钮 + 箭头（右侧），这边需要去阅读现在主窗口的格式是如何设计的，然后沿用

```

**设计要点：**
- 格式工具栏位于底部，但**不占整条**，而且没有分割线
- 收起时：只显示一个小箭头（‹），宽度约14px，靠右对齐
- 展开时：显示 B/I/U 三个按钮 + 箭头（›），整体居右
- 按钮垂直排列（与主窗口一致）
- 点击箭头切换展开/收起状态

**不包含的按钮：**
- ❌ 新建按钮（＋）
- ❌ 列表按钮（▲/▼）
- ❌ 回收站按钮（🗑）

### 数据同步机制

```
┌─────────────────┐         ┌──────────────┐
│   主窗口便签     │         │ 独立便签窗口  │
│                 │         │              │
│  [编辑内容]     │◄───────►│  [编辑内容]  │
│       ↓         │  实时同步 │       ↓      │
│  自动保存(1s)   │         │  自动保存(1s) │
│       ↓         │         │       ↓      │
└───────┬─────────┘         └──────┬───────┘
        │                          │
        └──────────┬───────────────┘
                   ↓
           ┌───────────────┐
           │  NoteService  │
           │   (数据库)     │
           └───────────────┘
```

**同步规则：**
1. 两个窗口共享同一个 `Note` 对象（通过 `note_id` 关联）
2. 任一窗口编辑后触发自动保存（1秒延迟）
3. 保存到数据库后，另一窗口无需刷新（因为是同一对象）
4. 如果需要实时同步显示，可实现观察者模式

**删除处理：**
```
主窗口删除便签
    ↓
NoteService.delete(note_id)
    ↓
独立窗口检测到便签被删除
    ↓
显示提示："此便签已被删除"
    ↓
自动关闭独立窗口
```

## 总结对比

### 优化前 vs 优化后

| 功能点 | 优化前 | 优化后 |
|--------|--------|--------|
| **便签列表** | 占满整个页面，遮挡编辑区 | 悬浮卡片，与按钮同宽，不遮挡内容 |
| **底部栏** | 整条有背景色，视觉厚重 | 背景透明，只有按钮有背景，轻量化 |
| **便签使用** | 只能在主窗口查看/编辑 | 可拖出独立窗口，多任务参考 |
| **视觉层次** | 平面化，缺乏层次 | 悬浮+阴影，层次分明 |
| **灵活性** | 单窗口操作 | 支持多窗口，灵活组织 |
